    Next Steps¶

    The workbook is now structurally complete, with:

    • Preface
    • Introduction
    • Full workbook content (Part 1, Bridging Section, Part 2)
    • Activity templates

    The following steps will ensure the workbook is complete, consistent, and ready for publication.

    1. Develop the Glossary¶

    Create a glossary aligned with Material for MkDocs formatting that:

    • Defines key terms used throughout the workbook (e.g., theory of change, cognitive load, active learning, formative assessment, OER, adaptation, localisation, attribution, metadata)
    • Uses clear, concise, non-academic language
    • Supports reuse and accessibility for diverse audiences

    Note

    After creating the glossary, do a terminology consistency pass across the workbook.

    2. Find relevant visuals¶

    1. Training as a System (FOUNDATIONAL)¶

    Purpose¶

    Make “training as an intervention in a system” immediately visible.

    What it should show¶

    • learners, facilitators, institutions
    • resources
    • constraints
    • external influences
    • relationships (arrows)

    Search terms¶

    • systems thinking diagram simple nodes arrows
    • training system stakeholders diagram
    • ecosystem diagram people resources constraints

    Recommendation¶

    ➡️ Prefer: RECREATE (based on reused idea)
    Most existing diagrams are too generic or business-oriented.

    2. Theory of Change / Logic Model¶

    Purpose¶

    Clarify the flow:
    Inputs → Activities → Outputs → Outcomes → Impact

    What it should show¶

    • linear or semi-linear flow
    • clear separation between stages

    Search terms¶

    • theory of change diagram CC BY
    • logic model diagram simple boxes arrows
    • results chain diagram inputs outputs outcomes impact

    Recommendation¶

    ➡️ Reuse OR lightly adapt

    This is a standard model and widely available.
    The structure is consistent across sources.

    3. How Learning Works (Cognitive + Social)¶

    Purpose¶

    Make learning mechanisms visible:

    • cognitive load
    • retrieval
    • reinforcement
    • social learning

    What it should show¶

    • working memory vs long-term memory
    • feedback / retrieval loop
    • interaction layer (peers)

    Search terms¶

    • cognitive load diagram working memory long term memory
    • learning cycle retrieval practice diagram
    • social learning diagram interaction

    Recommendation¶

    ➡️ RECREATE (important)
    You combine multiple theories—existing diagrams won’t match your synthesis.

    4. Alignment Triangle (Constructive Alignment)¶

    Purpose¶

    Show relationship between:

    • outcomes
    • activities
    • assessment

    What it should show¶

    • triangle or 3-part relationship
    • mutual alignment

    Search terms¶

    • constructive alignment triangle diagram
    • learning outcomes activities assessment triangle
    • backward design triangle

    Recommendation¶

    ➡️ Reuse OR recreate (very easy to redraw)

    This is simple enough to standardise visually across your workbook.

    5. Co-Design Under Constraints¶

    Purpose¶

    Make visible:

    • participants
    • facilitators
    • constraints
    • decision space

    What it should show¶

    • overlapping roles
    • constraints acting on system
    • trade-offs

    Search terms¶

    • co design process diagram stakeholders constraints
    • participatory design diagram simple
    • design constraints diagram influence

    Recommendation¶

    ➡️ RECREATE

    Your framing (ethics + constraints + participation) is specific.

    • feedback

    Search terms¶

    • lesson activity structure diagram
    • learning activity framework diagram

    Recommendation¶

    ➡️ RECREATE

    This should align exactly with your templates.

    7. Practice → Feedback → Iteration Loop¶

    Purpose¶

    Reinforce learning cycle:

    • attempt
    • feedback
    • revision

    What it should show¶

    • circular loop
    • progression (v0 → v1 → v2)

    Search terms¶

    • iteration loop diagram improvement cycle

    ➡️ Reuse OR recreate

    8. Assessment vs Evaluation¶

    Purpose¶

    Clarify distinction:

    • assessment = learning
    • evaluation = impact

    What it should show¶

    • two parallel or layered processes
    • different focus and timing

    Search terms¶

    • assessment vs evaluation diagram education
    • formative vs summative vs impact diagram
    • training evaluation levels diagram

    Recommendation¶

    ➡️ RECREATE (recommended)
    Most diagrams are overly academic or confusing.

    9. Reuse – Adapt – Create Decision Model¶

    Purpose¶

    Support decision-making:

    • reuse
    • adapt
    • create

    What it should show¶

    Search terms¶

    Recommendation¶

    ➡️ RECREATE (high value)
    This is central to your workbook’s argument.

    3. Complete Citation Information¶

    Update the Citation section in the preface:

    • Provide a recommended citation format
    • Include:
    • authors
    • year
    • title
    • version (if applicable)
    • DOI (if publishing on Zenodo)

    4. Decide on Reference Strategy¶

    Confirm how references should be presented:

    • Keep section-level “Further reading” only
      OR
    • Add a consolidated bibliography at the end

    Tip

    A consolidated bibliography improves usability for academic and policy audiences.

    5. Validate Activity Templates¶

    Conduct a full template audit:

    • Confirm all referenced templates exist
    • Ensure numbering and naming are consistent
    • Verify all links resolve correctly
    • Check that each template aligns with the section that references it

    6. Terminology Consistency Pass¶

    After the glossary is complete:

    • Standardise key terms across the workbook
    • Resolve inconsistencies such as:
    • co-design vs co-creation
    • outputs / outcomes / impact
    • assessment vs evaluation
    • reuse / adapt / create
    • sharing vs publishing
    • localisation vs adaptation

    7. MkDocs Structure and Navigation¶

    Prepare the workbook for publishing:

    • Finalise mkdocs.yml navigation structure
    • Ensure logical section ordering
    • Check:
    • previous/next navigation
    • internal links
    • anchor links
    • image paths
    • admonition rendering

    8. Final Editorial Quality Assurance¶

    Conduct a full end-to-end review:

    • Remove duplication where unintended
    • Ensure consistent tone and clarity
    • Validate heading hierarchy (##, ###)
    • Check formatting consistency
    • Confirm readability in Material for MkDocs

    9. Prepare for Publication¶

    Finalize outputs for distribution:

    • Generate versioned release (e.g., v1.0)
    • Upload to Zenodo (if applicable)
    • Confirm license visibility (CC BY 4.0)
    • Prepare landing page (GitHub Pages / MkDocs site)

    Outcome¶

    After completing these steps, the workbook will be:

    • structurally complete
    • internally consistent
    • pedagogically aligned
    • technically ready for publication
    • reusable and adaptable across contexts
