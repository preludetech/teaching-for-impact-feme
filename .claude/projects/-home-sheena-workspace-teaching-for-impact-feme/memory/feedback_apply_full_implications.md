---
name: Apply full implications of feedback
description: When the user gives a reason for a change, apply all consequences — don't make them spell out each one separately.
type: feedback
---

When the user explains *why* something should change (e.g. "they already did pre-work so they won't need as much time"), apply every consequence of that reasoning in the same edit — don't just update the description and leave the timing unchanged.

**Why:** The user had to repeat themselves and got frustrated when only part of the implication was acted on.

**How to apply:** After making the directly requested change, check whether the same reasoning affects nearby fields (time allocations, instructions, grouping, etc.) and update those too.
