# Visual Aid Style Guide (v1 — Integrated)

This style guide defines how to design, adapt, and use visual aids in the workbook:

**"Teaching for Impact: Designing Effective & Open Training"**

Visual aids must support learning, decision-making, and real-world application — not decoration.

---

## 1. Design Principles

### When to use visuals
Use visuals when they:
- explain processes or workflows
- show relationships between concepts
- support decisions
- simplify complexity
- improve recall

### High-value visuals
A high-value visual:
- communicates one core idea
- reduces text
- is understandable at a glance
- supports application
- aligns with a specific learning objective

### What to avoid
- decorative visuals
- overcrowded diagrams
- inconsistent styles
- unnecessary colour variation
- text-heavy visuals

---

## 2. Visual Hierarchy

### Purpose
Ensure every visual has a clear focal point and reading path.

### Rules
- 1 primary element (dominant)
- 2–3 secondary groups
- supporting elements only

### How to create hierarchy
- size
- colour intensity
- position (centre = highest importance)

### Avoid
- equal-sized elements
- multiple focal points

---

## 3. Visual Argument

### Rule
Before designing:

This visual shows that: ___________

If you cannot complete this clearly → redesign.

---

## 4. Element Reduction

### Rules
- remove redundancy
- reduce elements by 30–50% after first draft

### Test
If removing something does not break meaning → remove it

---

## 5. Colour System

### Talarify palette
- #1d647d — primary teal
- #638fa8 — muted blue
- #9cb5cb — light blue-grey
- #507d8e — teal-grey
- #d46852 — accent coral
- #004b6c — dark teal

### Colour roles
- primary → main structure
- secondary → supporting elements
- background → grouping
- neutral → connectors
- accent → highlights

### Rules
- max 3–4 colours
- consistent meaning
- accent used sparingly
- ensure contrast

---

## 6. Typography

### Fonts
- Inter
- Source Sans Pro
- Arial

### Rules
- short phrases only
- avoid sentences
- strong contrast
- simple hierarchy

---

## 7. Spatial Composition

### Rules
- left → actors / inputs
- centre → intervention
- right → system / outcomes

- use whitespace for grouping
- avoid heavy containers

---

## 8. Layout System

### Flow layout
- left-to-right
- evenly spaced
- minimal crossing

### Cycle layout
- clockwise
- clearly defined stages

### Triangle layout
- balance between three elements

### System map
- centre = intervention
- left = actors
- right = conditions
- minimal clutter

### Decision tree
- clear branching
- limited depth

---

## 9. Shapes and Components

### Standard shapes
- Rounded rectangle → default
- Rectangle → structured steps
- Circle → cycles
- Diamond → decisions
- Container → grouping (use sparingly)

### Rules
- consistent corners
- simple fills
- maintain spacing

---

## 10. Arrows and Connections

### Arrow types
- solid → direct effects
- dashed → indirect/system effects
- broken → routing around constraints

### Rules
- max 3–6 arrows
- arrows must reflect causal logic
- avoid connecting everything

### Direction
- must match meaning (cause → effect)

---

## 11. Label Quality

### Rules
- ≤ 6 words
- precise
- action-oriented

### Avoid
- vague phrases
- generic statements

### Prefer
- builds capability and agency
- enables change within and beyond constraints

---

## 12. Feedback & System Effects

### Rules
- feedback represents effects of the intervention
- distinguish:
  - direct (people)
  - indirect (system)

- route arrows outside text zones
- avoid overlap

---

## 13. Icon Usage

### Use icons when
- they improve recognition
- support scanning

### Rules
- single icon set
- simple icons
- always label

---

### Background
- white default
- light grouping optional

### Formats

- PNG fallback

- centre: intervention

## 18. Quality Checklist

- clear in <8 seconds
- one focal point
- no overlap
- minimal text
- consistent design
- reusable

---

## 19. Polish Pass

Before finalising:
- align to grid
- fix spacing
- check arrow routing
- remove unnecessary elements
- confirm hierarchy is clear
